oreilly
 
* Major upgrade to dump file format (object save/load) -- now saves type and name of object
at the end of a path pointer, and these are used during loading to find the right guy (in
addition to the path) -- prevents all these crazy problems when loading into a 
project with slightly different orders or whatever of programs.
* Also fixed the path token stuff -- it now saves full paths as tokens and uses hash
table lookup for speed, making files much more robust for merging with svn etc.
* Updated all the std programs -- now have new save format.
* NOTE: this save format is INCOMPATIBLE with prior versions -- once you save in
this new format, file will NOT load in earlier versions.  there is a flag
in preferences to save in old format but it should only be used if absolutely
necessary.  embrace the future!
