oreilly
 
* Bumped version to 8.0.3 to auto-update projects for new column sizing stuff..
* datatable colum widths all working great -- key issue was that we were calling the
"imperative" resize method, instead of just setting the flag, and allowing Qt to take
care of everything for us automatically.. Meanwhile, the iDataTableEditor had a whole
redundant loop of column width setting that was messing with things (unbeknownst to me
at least), and then, insult to injury, the iDataTableColHeaderView::resizedSection call
was being called for *ALL* resizes (including programmatically triggered ones) and
re-setting flags and widths!  It is now all much simpler!
* replaced DataCol bool with a flag for SIZE_TO_CONTENTS, and initialize all existing,
  < 8.0.3 projects to use this SIZE_TO_CONTENTS flag and set a default fallback width based on
  GridColDisplayWidth (renamed that from previously ambiguous displayWidth) and name..
* add a max_col_width to the DataTable -- defaults to 50 -- BUT, if a user resizes a colunn
wider than that, it is automatically increased.  Otherwise, there is NO way to ever get
a column wider than this max..
:* MUST use the gui menu Set Column Width thing to set a col width > the current max --
  gui will not let you do that..
:* This is also true for columns that are auto-sizing (i.e., all of them now) -- gui won't
   let you resize an auto-sized column.  So use the gui method (or settings on the DataCol)
* ClusterRun tables initialized to row_height=2 for projects < 8.0.3 as new default..
* initialize extended select mode in iTextEdit to false -- was randomly on and causing
things to be cut etc..  we should look for a clang tool that figures out if you never
intialize a class member -- seems like that is a major source of bugs..
* RandomSpec is actually fully redundant with Random now. Oh well.  Not worth fixing.
* NetMonItem don't check config for off items
* UnitSpec duplicate was causing a weird error for bias spec in Bp -- fixed CheckObjType..
not sure how that ever worked properly before..
* fixed bad connect name in svn browser
