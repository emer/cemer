2013-07-21
 
rohrlich
 
Added a user settable flag "save_as_only" on project that disables "save" action but keeps "save as" action available. When "save as" is executed the file name has "_copy" appended to the file name, for example, LeabraStd.proj becomes LeabraStd_copy.proj. This should keep users from accidently overwriting files that are demos or tutorials not meant to be overwritten. This isn't truely "read-only" because the user can change the name in the save dialog but then the flag on the project is user settable anyway. The "save_as_only" flag is set to false on the copy so the copy can be "saved".
