2012-03-14
 
oreilly
 
* css parsing of [..] now always generates a Matrix object -- supports Matlab-style initializer syntax:
** a = [1,2,3; 4,5,6]; = matrix with 2 rows x 3 columns 
** a = [1:5:1]; = 2d matrix with inner dim = 3, used for slicing operations
** a = [:]; = above with 0:-1:1 defaults
** a = [::-1] = reverse order, etc etc
* python-style slicing and indexing for List, String, and soon Matrix, Array objects
** new Variant taBase::Elem(Variant idx) function that parses idx as a matrix and 
   does the right thing in different taBase subclasses (only List impl now)
** in case where Elem returns multiple items, a new List object (clone token) is made,
   filled with Link pointers to original objects, and a "el_view" matrix is installed
   with active indices or a byte mask -- iterator and display routines use this view
   to only expose selected items from original list -- same semantics as python.
** added new generic TA iterator that uses view filtering -- may require further tweaking
   for Matrix class
** had to add cssTA_Base ref of ptr when ptr_cnt = 0 and we don't own -- this happens
   and wasn't supported before -- oversight.  hopefully doesn't cause new issues.. :)
** new cssTA_Matrix obj to support matrix-specific operators etc
** coming: datatable and taGroup will support multidimensional indexing semantics -- first
   level is cols for datatable, subgroups for taGroup, next levels are items within, etc.
** fixed css auto-type to unspecified objects -- a = 1; a = "foo"; a = matrix per above all
   give var appropriate type automatically -- had extra 'term' in yacc syntax -- weird.
** matrix index model will automatically support matlab-style "find" etc..
** css will auto-detect matrix args to functions that take an element, and unroll using
   iterator across all elements -- not in place yet -- will work with user-defined funs too
* Version bumped to 5.3.3 to indicate newness
* work in progress -- compiles and works and should work with existing code at this point,
but not very thoroughly tested and more to be done -- not recommmended to use..
